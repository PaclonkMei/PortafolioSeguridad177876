MECANISMOS DE DEFENSA EN RED

IMPLEMENTACIÓN DE REGLAS DE FIREWALL (IPTABLES)

/ETC/NET/TOPOLOGY_MAP.PNG

La siguiente configuración asume una arquitectura de red con una zona desmilitarizada (DMZ) donde residen los servidores públicos.

[ INTERNET (0.0.0.0/0) ]
|
[ FIREWALL ]
|
[ DMZ (192.1.2.0/24) ]
|-- MAIL SERVER (192.1.2.10)
|-- WEB SERVER (192.1.2.11)
/BIN/HARDENING_RULES.SH

Script de Configuración

Traducción de requerimientos a reglas de sintaxis Iptables.

# 1. Establecer una política restrictiva (Default DROP) iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT # Generalmente se permite salida, o se restringe según necesidad # 2. Permitir el tráfico de conexiones ya establecidas (Stateful Inspection) iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -A FORWARD -m state --state ESTABLISHED,RELATED -j ACCEPT # 3. Aceptar tráfico DNS (TCP/UDP) saliente de la red local # Nota: DNS usa principalmente UDP puerto 53, TCP para transferencias de zona o respuestas grandes. iptables -A FORWARD -p udp --dport 53 -s 192.1.2.0/24 -j ACCEPT
iptables -A FORWARD -p tcp --dport 53 -s 192.1.2.0/24 -j ACCEPT # 4. Aceptar correo entrante (SMTP/IMAP/POP3) al Servidor de Correo iptables -A FORWARD -p tcp -d 192.1.2.10 -m multiport --dports 25,110,143,465,993,995 -j ACCEPT # 5. Permitir correo saliente (SMTP) desde el Servidor de Correo hacia Internet iptables -A FORWARD -p tcp -s 192.1.2.10 --dport 25 -j ACCEPT # 6. Aceptar conexiones HTTP/HTTPS desde Internet al Servidor Web iptables -A FORWARD -p tcp -d 192.1.2.11 --dport 80 -j ACCEPT
iptables -A FORWARD -p tcp -d 192.1.2.11 --dport 443 -j ACCEPT # 7. Permitir tráfico HTTP desde la red local hacia Internet (Navegación) iptables -A FORWARD -p tcp -s 192.1.2.0/24 --dport 80 -j ACCEPT
iptables -A FORWARD -p tcp -s 192.1.2.0/24 --dport 443 -j ACCEPT
