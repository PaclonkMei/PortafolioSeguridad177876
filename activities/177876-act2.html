ANÁLISIS DE SERVICIOS X.800 Y RFC 4949

FUNDAMENTOS DEL HACKING ÉTICO: ESCENARIOS DE INCIDENTES

/DOCS/00_INTRODUCCION.TXT
OBJETIVO GENERAL:
Analizar escenarios de incidentes de seguridad informática mediante la aplicación de los seis servicios de seguridad definidos en el ITU-T X.800 y el uso preciso de la terminología del RFC 4949.

Marco Conceptual

RFC 4949 (Glosario): Estándar global de la IETF para la jerga en Ciberseguridad. Define riesgos, amenazas y vulnerabilidades para mejorar la claridad en la documentación técnica y el debate.

ITU-T X.800 (Arquitectura de Seguridad OSI): Marco conceptual para implementar seguridad en redes. Complementa el modelo OSI y sirve de base para normas como ISO 27000. Se enfoca en autenticación, control de acceso, confidencialidad, integridad, no repudio y disponibilidad.

CASE_01: RANSOMWARE LOCKBIT [HIGH SEVERITY]

Cifrado masivo de servidores tras exfiltración de información. Compromiso simultáneo de CIA (Confidencialidad, Integridad, Disponibilidad).

Servicios X.800Autenticación, Control de accesos, Confidencialidad
Definiciones RFC 4949
  • Authentication: Verificación de identidad declarada.
  • Credential Compromise: Exposición de claves a acceso no autorizado.
  • One-way Encryption: Transformación irreversible (ataque).
Vector de AtaqueData Breach + Cifrado Malicioso.
ImpactoPérdida de datos, daño infraestructura, compromiso usuarios.
ControlRespaldos inmutables, MFA, detección de accesos remotos.
CASE_02: CLOUD MISCONFIGURATION [MED SEVERITY]

Bases de datos públicas por error de configuración en la nube. No hubo intrusión activa sofisticada.

Servicios X.800Confidencialidad de datos
Definiciones RFC 4949
  • Exposure: Condición donde info sensible queda accesible.
  • Misconfiguration: Configuración incorrecta que introduce vulnerabilidad.
  • Unauthorized Access: Acceso a recurso sin permiso.
Vector de AtaqueConfiguración incorrecta de ACLs en almacenamiento.
ImpactoDivulgación de info sensible, daño reputacional y legal.
ControlValidación de configs, principio de mínimo privilegio.
CASE_03: SUPPLY CHAIN ATTACK [CRITICAL]

Proveedor legítimo de software comprometido distribuyó malware en actualizaciones firmadas.

Servicios X.800Integridad de datos
Definiciones RFC 4949
  • Integrity: Datos no modificados sin autorización.
  • Supply Chain Attack: Compromiso de elemento confiable (proveedor).
  • Malicious Code: Software diseñado para dañar.
Vector de AtaqueActualización firmada legítimamente con Payload malicioso.
ImpactoViolación de confianza, acceso masivo a clientes.
ControlVerificación independiente de hashes, monitoreo de comportamiento.
CASE_04: PHISHING & PERSISTENCE [HIGH SEVERITY]

Atacantes obtuvieron credenciales válidas y mantuvieron acceso por meses sin levantar alertas.

Servicios X.800Autenticación, Control de Acceso
Definiciones RFC 4949
  • Credential Compromise: Robo de credenciales válidas.
  • Authentication Failure: Falla conceptual (el sistema aceptó al atacante).
Vector de AtaquePhishing + Uso prolongado de accesos legítimos.
ImpactoAcceso persistente, pérdida de control de sistemas.
ControlMFA (FIDO2), análisis de comportamiento de usuarios (UEBA).
CASE_05: BACKUP DESTRUCTION [CRITICAL]

Eliminación o cifrado de respaldos antes del ataque principal para impedir la recuperación.

Servicios X.800Disponibilidad, Integridad
Definiciones RFC 4949
  • Availability Attack: Acción deliberada para impedir acceso.
  • Data Destruction: Eliminación intencional de datos.
Vector de AtaqueBorrado de backups previos al cifrado.
ImpactoPérdida total de capacidad de recuperación. Catastrófico.
ControlRespaldos Offline (Air-gapped) o inmutables.
CASE_06: INSIDER THREAT [MED SEVERITY]

Empleado con acceso legítimo extrajo y vendió bases de datos. Abuso de confianza.

Servicios X.800Confidencialidad, Control de Acceso
Definiciones RFC 4949
  • Insider Threat: Amenaza originada por entidad interna autorizada.
  • Excessive Privilege: Asignación de permisos superiores a lo necesario.
Vector de AtaqueAbuso deliberado de privilegios legítimos.
ImpactoExfiltración masiva, pérdida de confianza.
ControlSegregación de funciones, DLP (Data Loss Prevention).
CASE_07: LOG ALTERATION [HIGH SEVERITY]

Cifrado o modificación de logs para impedir análisis forense y borrar huellas.

Servicios X.800Integridad, No Repudio
Definiciones RFC 4949
  • Non-Repudiation: Garantía de no poder negar autoría.
  • Evidentiary Integrity: Valor probatorio de la evidencia digital.
  • Audit Trail: Registro cronológico de eventos.
Vector de AtaqueModificación de registros del sistema (logs).
ImpactoImposibilidad de forense, pérdida de evidencia legal.
ControlEnvío de logs a servidor remoto seguro (SIEM) en tiempo real.
CASE_08: OPERATIONAL FAILURE [HIGH SEVERITY]

Actualización defectuosa (tipo CrowdStrike) provocó caída global de servicios.

Servicios X.800Disponibilidad
Definiciones RFC 4949
  • Availability: Propiedad de estar accesible cuando se requiere.
  • Operational Failure: Falla por error interno (no ataque malicioso).
Vector de AtaqueFalla operacional interna (Update sin testing).
ImpactoInterrupción simultánea crítica a escala global.
ControlPruebas en Staging, mecanismos de Rollback automáticos.
CASE_09: SITE CLONING (PHISHING) [MED SEVERITY]

Replica de sitios oficiales para engañar ciudadanos. Suplantación de identidad.

Servicios X.800Autenticación, Confidencialidad
Definiciones RFC 4949
  • Masquerade: Entidad que se hace pasar por otra legítima.
  • Phishing: Ingeniería social mediante engaño.
Vector de AtaqueSitios web falsificados (Typosquatting).
ImpactoRobo de credenciales y datos personales.
ControlAutenticación de dominio, concientización de usuarios.
CASE_10: DESTRUCTIVE ATTACK [CRITICAL]

Acciones destructivas para borrar sistemas completos tras exfiltración (Wiper Malware).

Servicios X.800Confidencialidad, Integridad, Disponibilidad
Definiciones RFC 4949
  • Destructive Attack: Objetivo principal es daño irreversible.
  • Integrity: Sistemas destruidos de manera no autorizada.
Vector de AtaqueExfiltración seguida de Wiping (Borrado seguro).
ImpactoPérdida irreversible, colapso operativo total.
ControlSegmentación crítica, plan de recuperación de desastres (DRP).
