IMPLEMENTACIÓN IPSEC VPN

CONFIGURACIÓN DE TÚNELES SEGUROS EN CISCO PACKET TRACER

/DOCS/01_INTRO.TXT

Introducción

La protección de datos en redes públicas suele ser complicada debido al libre acceso, lo que permite que los datos puedan ser modificados o suplantados en tránsito. Para mitigar esto, se utiliza IPSec (Internet Protocol Security), un conjunto de protocolos de la capa de red que establece canales seguros mediante mecanismos criptográficos de autenticación, integridad y confidencialidad.

En esta práctica se implementa una VPN IPSec en Packet Tracer. El objetivo es establecer una comunicación segura entre dos redes LAN (R1 y R3) a través de un ISP simulado, asegurando que el tráfico esté cifrado antes de atravesar la red pública.

Desarrollo: Enrutamiento Base

Se configuraron las IPs y el enrutamiento estático para establecer conexión de extremo a extremo antes de aplicar seguridad.

! --- Configuración Router 1 (R1) --- R1(config)# int G0/0 R1(config-if)# ip add 209.165.100.1 255.255.255.252 R1(config-if)# no shutdown R1(config-if)# int G0/1 R1(config-if)# ip add 192.168.1.1 255.255.255.0 R1(config-if)# no shutdown R1(config-if)# exit R1(config)# ip route 0.0.0.0 0.0.0.0 209.165.100.2 ! --- Configuración ISP --- ISP(config)# int g0/0 ISP(config-if)# ip add 209.165.100.2 255.255.255.252 ISP(config-if)# no shutdown ISP(config-if)# int g0/1 ISP(config-if)# ip add 209.165.200.2 255.255.255.252 ISP(config-if)# no shutdown ISP(config-if)# exit ISP(config)# ip route 192.168.1.0 255.255.255.0 209.165.100.1 ISP(config)# ip route 192.168.3.0 255.255.255.0 209.165.200.1 ! --- Configuración Router 3 (R3) --- R3(config)# int G0/0 R3(config-if)# ip add 209.165.200.1 255.255.255.252 R3(config-if)# no shutdown R3(config-if)# int G0/1 R3(config-if)# ip add 192.168.3.1 255.255.255.0 R3(config-if)# no shutdown R3(config-if)# exit R3(config)# ip route 0.0.0.0 0.0.0.0 209.165.200.2

Implementación de Túnel IPSec

1. Habilitar Licencia de Seguridad y Tráfico Interesante
Fue necesario habilitar el módulo `securityk9` para permitir cifrado. Luego, se definió mediante ACL 100 qué tráfico debe ser protegido por la VPN.

R1(config)# license boot module c1900 technology-package securityk9 R1(config)# access-list 100 permit ip 192.168.1.0 0.0.0.255 192.168.3.0 0.0.0.255 ! En R3 se invierte el origen y destino: R3(config)# access-list 100 permit ip 192.168.3.0 0.0.0.255 192.168.1.0 0.0.0.255

2. Fase 1 - ISAKMP (Negociación)
Establecimiento del canal seguro intercambiando parámetros de cifrado (AES-256) y la llave precompartida (PSK: santrum).

R1(config)# crypto isakmp policy 10 R1(config-isakmp)# encryption aes 256 R1(config-isakmp)# authentication pre-share R1(config-isakmp)# group 5 R1(config-isakmp)# exit R1(config)# crypto isakmp key santrum address 209.165.200.1

3. Fase 2 y 3 - Transform Set & Crypto Map
El Transform Set define cómo se protegerán los datos (ESP-AES). El Crypto Map vincula el peer remoto, los algoritmos y la ACL.

! Transform Set R1(config)# crypto ipsec transform-set R1->R3 esp-aes 256 esp-sha-hmac ! Crypto Map R1(config)# crypto map IPSEC-MAP 10 ipsec-isakmp R1(config-crypto-map)# set peer 209.165.200.1 R1(config-crypto-map)# set transform-set R1->R3 R1(config-crypto-map)# set pfs group5 R1(config-crypto-map)# set security-association lifetime seconds 86400 R1(config-crypto-map)# match address 100 R1(config-crypto-map)# exit ! Fase 4: Aplicar a la interfaz WAN (G0/0) R1(config)# int g0/0 R1(config-if)# crypto map IPSEC-MAP *Jan 3 07:16:26.785: %CRYPTO-6-ISAKMP_ON_OFF: ISAKMP is ON

La implementación de la VPN IPSec permitió conectar ambas redes privadas a través del ISP como si fueran una sola red lógica, garantizando confidencialidad, integridad y autenticación del tráfico. El túnel se construye dinámicamente cuando aparece tráfico interesante, demostrando que la seguridad no solo depende del enrutamiento, sino de mecanismos criptográficos extremo a extremo.
